# 題目構想 (呂依裴C112181137)

## 題目名稱

基於 Docker 容器化技術之社群媒體輿情情緒監測與分析平台

## 為什麼對這個題目有興趣？

在數位行銷時代，了解大眾對產品或事件的真實看法至關重要。傳統的人工瀏覽評論不僅耗時，且容易產生主觀偏誤。我希望透過建立一個自動化的監測平台，利用 Python 的強大數據處理能力與 Keras 的 AI 情緒分析模型，將碎片化的社群文字轉化為可視化的數據指標。同時，透過 Docker 容器化技術，確保此工具能在不同環境中穩定部署，提升開發與營運的效率。

## 可能使用的資料來源

| 資料名稱 | 來源網址 | 資料格式 |
|---------|---------|---------|
|Twitter/X 評論數據集 | Kaggle (Sentiment140)| CSV|
|PTT/Dcard 模擬評論資料 | 自行模擬生成或 GitHub 開源 API| JSON / CSV|
|IMDB 影評大數據 | Keras 內建數據集|NumPy Array|

## 預計使用的技術

- [x] Python + Pandas 資料分析：用於清洗評論內容（去除雜訊、標點符號）。
- [x] Matplotlib / Seaborn 視覺化：繪製正負面情緒分佈的圓餅圖與趨勢圖。
- [x] Jupyter Notebook：用於前期開發模型與探索式資料分析（EDA）。
- [x] Keras 預訓練模型：使用 LSTM 或 BERT 模型進行文字情緒（正向/負向）分類。
- [x] Gradio 互動介面：提供快速測試單句情緒分析的 AI 演示區。
- [x] MySQL 資料庫：儲存使用者搜尋歷史、關鍵字與分析結果摘要。
- [x] Docker 容器部署：將 PHP 前端、Python 分析模組與 MySQL 分別打包部署。
- [x] 其他：__________Apache + PHP（作為主系統入口與歷史紀錄展示）。

## 預期成果

1.使用者介面：一個簡潔的網頁，使用者可輸入特定關鍵字啟動分析。
2.歷史紀錄系統：透過 PHP 呈現過去搜尋過的關鍵字與分析日期。
3.自動化分析報告：系統會自動執行 Python 腳本，分析該主題的情緒傾向，並動態產出圓餅圖（例如：70% 正向、30% 負向）。
4.容器化架構：提供一個 docker-compose.yml 檔案，只需一個指令即可在任何電腦啟動整個分析平台。
